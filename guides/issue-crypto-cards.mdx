This guide walks through the complete flow for issuing a Visa/Mastercard virtual card to a user: onboarding, KYC, wallet deployment, card creation, funding via USDC deposit, and securely displaying card details.

<Note>
  Test this flow in the Sandbox at `https://crypto-sandbox.yativo.com/api/v1/`. KYC checks are auto-approved in the Sandbox environment.
</Note>

## Prerequisites

- An authenticated Yativo Crypto API key
- User identity information for KYC (name, date of birth, address, government ID)

<Steps>

<Step title="Wait for KYC approval">

KYC review can be instant (automated) or take up to 24 hours for manual review. Poll the status endpoint or listen for the `kyc.approved` webhook event.

<CodeGroup>

```bash cURL
curl -X GET "https://crypto-api.yativo.com/api/v1/yativo-card/kyc-status?cardUserId=cus_01J2KJ3MNPQ7R4S5T6U7VCRD01" \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiJ9.eyJzdWIiOiJ1c3JfMDFIWVo..."
```

```typescript TypeScript
// Poll until approved or rejected
async function waitForKycApproval(cardUserId: string): Promise<string> {
  const maxAttempts = 60; // poll for up to 10 minutes
  let delay = 10000;

  for (let i = 0; i < maxAttempts; i++) {
    const status = await client.card.getKycStatus(cardUserId);

    if (status.kycStatus === "approved") return "approved";
    if (status.kycStatus === "rejected") {
      throw new Error(`KYC rejected: ${status.rejectionReason}`);
    }

    await new Promise((r) => setTimeout(r, delay));
  }

  throw new Error("KYC did not complete within timeout");
}

await waitForKycApproval("cus_01J2KJ3MNPQ7R4S5T6U7VCRD01");
```

```python Python
import time

def wait_for_kyc_approval(card_user_id: str) -> str:
    max_attempts = 60
    delay = 10.0

    for _ in range(max_attempts):
        status = client.card.get_kyc_status(card_user_id)

        if status.kyc_status == "approved":
            return "approved"
        if status.kyc_status == "rejected":
            raise Exception(f"KYC rejected: {status.rejection_reason}")

        time.sleep(delay)

    raise TimeoutError("KYC did not complete within timeout")

wait_for_kyc_approval("cus_01J2KJ3MNPQ7R4S5T6U7VCRD01")
```

</CodeGroup>

**KYC status values:**

| Status | Description |
|--------|-------------|
| `pending` | Under review |
| `approved` | User can proceed |
| `rejected` | KYC failed — reason in `rejectionReason` |
| `more_info_required` | Additional documents needed |

<Tip>
  Rather than polling, register a webhook for `kyc.status_changed` to get notified the moment the status updates.
</Tip>

</Step>

<Step title="Display card details securely with an ephemeral token">

Card details (full PAN, CVV, expiry) are sensitive and must never be stored or logged. Use the ephemeral token API to retrieve them for a short-lived display session.

<CodeGroup>

```bash cURL
curl -X POST "https://crypto-api.yativo.com/api/v1/yativo-card/cus_01J2KJ3MNPQ7R4S5T6U7VCRD01/cards/crd_01J2KL3MNPQ7R4S5T6U7VCRD88/ephemeral-token" \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiJ9.eyJzdWIiOiJ1c3JfMDFIWVo..." \
  -H "Content-Type: application/json" \
  -d '{
    "ttlSeconds": 300
  }'
```

```typescript TypeScript
// Backend: generate the ephemeral token
const ephemeral = await client.card.createEphemeralToken({
  cardUserId: "cus_01J2KJ3MNPQ7R4S5T6U7VCRD01",
  cardId: "crd_01J2KL3MNPQ7R4S5T6U7VCRD88",
  ttlSeconds: 300, // 5-minute window
});

// Return token to frontend (never log or store it)
res.json({ token: ephemeral.token, expiresAt: ephemeral.expiresAt });

// Frontend: use the token to fetch card details from Yativo's secure endpoint
// The token is single-use and expires after ttlSeconds
const cardDetails = await fetch("https://crypto-api.yativo.com/api/v1/yativo-card/reveal", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ token: ephemeral.token }),
}).then((r) => r.json());

// Display cardDetails.pan, cardDetails.cvv, cardDetails.expiry in your UI
// Never store or log these values
```

```python Python
# Backend: generate the ephemeral token
ephemeral = client.card.create_ephemeral_token(
    card_user_id="cus_01J2KJ3MNPQ7R4S5T6U7VCRD01",
    card_id="crd_01J2KL3MNPQ7R4S5T6U7VCRD88",
    ttl_seconds=300,
)

# Return token to frontend — never log or store it
return jsonify({"token": ephemeral.token, "expires_at": ephemeral.expires_at})
```

</CodeGroup>

**Ephemeral token response:**

```json
{
  "token": "eph_eyJhbGciOiJSUzI1NiJ9.eyJjYXJkSWQiOiJjcmRfMDFKMkts...",
  "expiresAt": "2026-03-26T12:13:00Z",
  "ttlSeconds": 300
}
```

<Warning>
  Never log, store, or transmit full card details (PAN, CVV) through your servers. Use the ephemeral token pattern to retrieve them directly in the browser or mobile app for display-only purposes.
</Warning>

## Physical Cards

To issue a physical card instead of virtual, set `"type": "physical"` in the create-card request and include shipping details:

```json
{
  "cardUserId": "cus_01J2KJ3MNPQ7R4S5T6U7VCRD01",
  "type": "physical",
  "label": "Alice Physical Card",
  "shipping": {
    "name": "Alice Martinez",
    "line1": "123 Main Street",
    "city": "San Francisco",
    "state": "CA",
    "postalCode": "94105",
    "country": "US"
  }
}
```

Physical card delivery typically takes 5–10 business days. The card arrives inactive and must be activated via `POST /yativo-card/{cardId}/activate`.
