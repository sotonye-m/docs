This guide walks through the full IBAN onboarding flow: checking available account options, submitting KYC, requesting an IBAN, and handling incoming bank transfer events via webhook.

With a Yativo standalone IBAN, you receive a dedicated account number (SEPA or SWIFT) that accepts bank transfers from anywhere in the world. Incoming funds are automatically converted to the crypto asset of your choice.

<Note>
  Test this flow in the Sandbox at `https://crypto-sandbox.yativo.com/api/v1/`. Sandbox KYC and IBAN activation are instant.
</Note>

## Activation Status Progression

```
[initiated] → [kyc_submitted] → [kyc_approved] → [iban_requested] → [active]
                                      ↓
                               [kyc_rejected] (resubmit or contact support)
```

<Steps>

<Step title="Check available IBAN options">

Retrieve the available IBAN types, supported currencies, and jurisdiction-specific requirements before starting onboarding.

<CodeGroup>

```bash cURL
curl -X GET https://crypto-api.yativo.com/api/v1/standalone-iban/options \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiJ9.eyJzdWIiOiJ1c3JfMDFIWVo..."
```

```typescript TypeScript
import { YativoCrypto } from "@yativo/crypto-sdk";

const client = new YativoCrypto({
  apiKey: process.env.YATIVO_API_KEY!,
  baseUrl: "https://crypto-api.yativo.com/api/v1/",
});

const options = await client.iban.getOptions();

console.log(options.availableTypes);
// ["sepa", "swift"]

console.log(options.supportedCurrencies);
// ["EUR", "GBP", "USD"]

console.log(options.kycRequirements);
// { individual: [...], business: [...] }
```

```python Python
from yativo_crypto import YativoCrypto

client = YativoCrypto(
    api_key=os.environ["YATIVO_API_KEY"],
    base_url="https://crypto-api.yativo.com/api/v1/",
)

options = client.iban.get_options()

print(options.available_types)       # ["sepa", "swift"]
print(options.supported_currencies)  # ["EUR", "GBP", "USD"]
```

<Step title="Initialize onboarding">

Start the onboarding session. This creates a record you will add KYC data to in the next step.

<CodeGroup>

```bash cURL
curl -X POST https://crypto-api.yativo.com/api/v1/standalone-iban/onboarding/init \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiJ9.eyJzdWIiOiJ1c3JfMDFIWVo..." \
  -H "Content-Type: application/json" \
  -d '{
    "accountType": "individual",
    "ibanType": "sepa",
    "currency": "EUR",
    "conversionAsset": "USDC",
    "label": "My EUR Receiving Account"
  }'
```

```typescript TypeScript
const onboarding = await client.iban.initOnboarding({
  accountType: "individual",
  ibanType: "sepa",
  currency: "EUR",
  conversionAsset: "USDC",
  label: "My EUR Receiving Account",
});

console.log(onboarding.onboardingId); // ibn_onb_01J2KS3MNPQ7R4S5T6U7VIBN01
console.log(onboarding.status);       // "initiated"
```

```python Python
onboarding = client.iban.init_onboarding(
    account_type="individual",
    iban_type="sepa",
    currency="EUR",
    conversion_asset="USDC",
    label="My EUR Receiving Account",
)

print(onboarding.onboarding_id)  # ibn_onb_01J2KS3MNPQ7R4S5T6U7VIBN01
print(onboarding.status)         # "initiated"
```

### KYC field requirements

**Individual accounts:**

| Field | Required | Notes |
|-------|----------|-------|
| `personal.firstName` | Yes | Legal first name |
| `personal.lastName` | Yes | Legal last name |
| `personal.dateOfBirth` | Yes | ISO 8601 (YYYY-MM-DD) |
| `personal.nationality` | Yes | ISO 3166-1 alpha-2 |
| `personal.countryOfResidence` | Yes | ISO 3166-1 alpha-2 |
| `address.line1` | Yes | Street address |
| `address.city` | Yes | |
| `address.postalCode` | Yes | |
| `address.country` | Yes | ISO 3166-1 alpha-2 |
| `identity.documentType` | Yes | `passport`, `national_id`, or `drivers_license` |
| `identity.documentNumber` | Yes | |
| `identity.issuingCountry` | Yes | |
| `identity.expiryDate` | Yes | Must be future date |
| `identity.frontImageUrl` | Yes | Pre-signed URL to uploaded image |
| `identity.backImageUrl` | If two-sided | Required for national IDs |
| `proofOfAddress.documentType` | Yes | `utility_bill`, `bank_statement`, or `government_letter` |
| `proofOfAddress.issuedDate` | Yes | Must be within last 3 months |
| `proofOfAddress.imageUrl` | Yes | Pre-signed URL to uploaded image |

<Warning>
  Document images must be uploaded to your own secure storage first, and the pre-signed URLs must be accessible by Yativo for the duration of the review process. URLs expire after 48 hours.
</Warning>

<Step title="Request the IBAN">

Once KYC is approved, submit the IBAN request to trigger account provisioning.

<CodeGroup>

```bash cURL
curl -X POST https://crypto-api.yativo.com/api/v1/standalone-iban/iban/request \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiJ9.eyJzdWIiOiJ1c3JfMDFIWVo..." \
  -H "Content-Type: application/json" \
  -d '{
    "onboardingId": "ibn_onb_01J2KS3MNPQ7R4S5T6U7VIBN01"
  }'
```

```typescript TypeScript
const request = await client.iban.requestIban({
  onboardingId: "ibn_onb_01J2KS3MNPQ7R4S5T6U7VIBN01",
});

console.log(request.status);   // "iban_requested"
console.log(request.ibanId);   // ibn_01J2KT3MNPQ7R4S5T6U7VIBN77
```

```python Python
request = client.iban.request_iban(
    onboarding_id="ibn_onb_01J2KS3MNPQ7R4S5T6U7VIBN01"
)

print(request.status)    # "iban_requested"
print(request.iban_id)   # ibn_01J2KT3MNPQ7R4S5T6U7VIBN77
```

</CodeGroup>

</Step>

<Step title="Poll for activation">

Once the IBAN is provisioned (typically within 1–2 business days), the status changes to `active` and the account details are available.

<CodeGroup>

```bash cURL
curl -X GET "https://crypto-api.yativo.com/api/v1/standalone-iban/status?ibanId=ibn_01J2KT3MNPQ7R4S5T6U7VIBN77" \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiJ9.eyJzdWIiOiJ1c3JfMDFIWVo..."
```

```typescript TypeScript
const ibanStatus = await client.iban.getStatus("ibn_01J2KT3MNPQ7R4S5T6U7VIBN77");

if (ibanStatus.status === "active") {
  console.log(ibanStatus.iban);          // DE89370400440532013000
  console.log(ibanStatus.bic);           // COBADEFFXXX
  console.log(ibanStatus.bankName);      // Commerzbank AG
  console.log(ibanStatus.accountName);   // Carlos Vega
  console.log(ibanStatus.currency);      // EUR
}
```

```python Python
iban_status = client.iban.get_status("ibn_01J2KT3MNPQ7R4S5T6U7VIBN77")

if iban_status.status == "active":
    print(iban_status.iban)          # DE89370400440532013000
    print(iban_status.bic)           # COBADEFFXXX
    print(iban_status.bank_name)     # Commerzbank AG
    print(iban_status.account_name)  # Carlos Vega
    print(iban_status.currency)      # EUR
```

<Step title="Receive bank transfers as webhook events">

When a bank transfer arrives to the IBAN, Yativo fires an `iban.transfer.received` event, followed by `iban.transfer.converted` once the conversion to USDC is complete.

```typescript TypeScript (webhook handler)
case "iban.transfer.received": {
  const { ibanId, amount, currency, senderName, senderIban, reference } = event.data;

  // Notify user — conversion in progress
  console.log(`EUR ${amount} received from ${senderName}`);
  break;
}

case "iban.transfer.converted": {
  const {
    ibanId,
    receivedAmount,
    receivedCurrency,
    convertedAmount,
    convertedAsset,
    exchangeRate,
    txHash,
  } = event.data;

  // Credit user's crypto balance
  await db.balances.credit(userId, convertedAsset, convertedAmount);

  console.log(
    `Converted EUR ${receivedAmount} → ${convertedAmount} ${convertedAsset} at rate ${exchangeRate}`
  );
  break;
}
```
