## Option A — API Key (Recommended, No Registration Needed)

The fastest way to get started. Use the **shared sandbox API key** to generate a bearer token instantly.

```bash
curl -X POST 'https://crypto-sandbox.yativo.com/api/auth/token' \
  -H 'Content-Type: application/json' \
  -d '{
    "api_key": "yativo_e072b3ef7e30fcb420183aa86ddd8452abd28a9ac8f5d04d",
    "api_secret": "df8da4d6b855c7e89bd3b4216dd0a1f4b30de1963c9bff0dcd666b15a0f836a5"
  }'
```

```json Response
{
  "success": true,
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "expires_at": "2026-03-28T11:00:00.000Z",
  "scopes": ["read", "write", "transactions"]
}
```

Use the `access_token` as `Authorization: Bearer <token>` on all requests. Tokens expire in 60 minutes — call `/auth/token` again to get a fresh one. **The API key itself never expires.**

<Warning>
  This is a **shared public sandbox key**. Anyone with this key can read and write to the shared sandbox account. For private testing, [create your own sandbox account](#option-b--register--log-in) and generate a personal API key.
</Warning>

## Option B — Register & Log In

If you want a private sandbox account with isolated data:

### Step 1 — Register

```
POST https://crypto-sandbox.yativo.com/api/auth/registration
```

```bash
curl -X POST 'https://crypto-sandbox.yativo.com/api/auth/registration' \
  -H 'Content-Type: application/json' \
  -d '{
    "email": "you@example.com",
    "password": "YourPassword123!",
    "first_name": "Test",
    "last_name": "User"
  }'
```

```json Response
{
  "status": true,
  "message": "Registration successful. Please verify your email.",
  "data": {
    "userId": "usr_6615c3a1e55d9ff7bc0a1234",
    "email": "you@example.com"
  }
}
```

### Step 2 — Log In

```
POST https://crypto-sandbox.yativo.com/api/auth/sign-in
```

```bash
curl -X POST 'https://crypto-sandbox.yativo.com/api/auth/sign-in' \
  -H 'Content-Type: application/json' \
  -d '{
    "email": "you@example.com",
    "password": "YourPassword123!"
  }'
```

### Step 3 — Verify OTP

```
POST https://crypto-sandbox.yativo.com/api/auth/otp-verification
```

```bash
curl -X POST 'https://crypto-sandbox.yativo.com/api/auth/otp-verification' \
  -H 'Content-Type: application/json' \
  -d '{
    "otp": "847291",
    "email": "you@example.com"
  }'
```

```json Response
{
  "status": true,
  "message": "Login successful",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "userId": "usr_6615c3a1e55d9ff7bc0a1234",
      "email": "you@example.com"
    }
  }
}
```

### Step 4 — Create Your Own API Key

Once logged in, create a permanent API key so you never have to go through the OTP flow again:

```bash
curl -X POST 'https://crypto-sandbox.yativo.com/api/apikey/generate' \
  -H 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{"name": "My Sandbox Key"}'
```

Store the returned `api_key` and `api_secret`. You can then use Option A above with your own private credentials.

## Refresh a Token

```
POST https://crypto-sandbox.yativo.com/api/auth/refresh-token
```

```bash
curl -X POST 'https://crypto-sandbox.yativo.com/api/auth/refresh-token' \
  -H 'Content-Type: application/json' \
  -d '{"refreshToken": "YOUR_REFRESH_TOKEN"}'
```

```json Response
{
  "status": true,
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

## Using the Access Token

Include the token on every authenticated request:

```bash
curl -X GET 'https://crypto-sandbox.yativo.com/api/accounts/get-accounts' \
  -H 'Authorization: Bearer YOUR_ACCESS_TOKEN'
```
